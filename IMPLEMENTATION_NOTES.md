# PaveRate Enhancements Implementation

This document describes the three enhancements implemented for PaveRate.

## 1. Light/Dark Mode Toggle

### Files Created/Modified:
- **Created:** `src/lib/stores/theme.svelte.ts` - Theme preference store with localStorage persistence
- **Created:** `src/lib/components/ThemeToggle.svelte` - Sun/moon icon toggle button
- **Modified:** `src/lib/config/paverate.yaml` - Added separate light and dark theme token sets
- **Modified:** `src/lib/config/index.ts` - Updated TypeScript interfaces for ThemeSet
- **Modified:** `src/routes/+layout.svelte` - Integrated theme system with reactive theme switching
- **Modified:** `src/routes/+page.svelte` - Added ThemeToggle to header
- **Modified:** `src/app.css` - Added data-theme attribute support and topbar-content class

### Features:
- Default is dark mode (preserves existing UX)
- Light mode uses warm cream/off-white background (#f5f1e8) with strong dark text for outdoor visibility
- Preference stored in localStorage
- Reactive theme switching via Svelte 5 runes
- Sun icon in dark mode, moon icon in light mode
- 48x48px touch target for glove-friendly interaction

### Light Mode Colors:
- Background: `#f5f1e8` (warm off-white)
- Surface: `#ffffff`
- Border: `#c9bfa8` (strong contrast)
- Text: `#1a1614` (strong dark)
- Accent: `#d4a42c` (darker yellow)

## 2. GDOT Reference Tables Page

### Files Modified:
- **Modified:** `src/routes/reference/+page.svelte` - Complete rebuild from placeholder

### Features:
- Three main tables with mobile-friendly horizontal scroll:
  - **Table 2:** Tack coat application rates (anionic/cationic emulsified asphalt)
  - **Table 4:** Lift thickness vs minimum temperature
  - **Table 5:** Mix type min/max layer and total thickness
- Mini table-of-contents at top with anchor links
- All data transcribed from `docs/GDOT_TABLES.md`
- Includes footnotes from Table 5
- Section 400.3.05.E weather limitations for OGFC/PEM
- Back button and theme toggle in header
- Responsive design with hover states
- Professional typography and spacing

## 3. PDF Proof/Export

### Files Created:
- **Created:** `src/lib/utils/pdf-export.ts` - PDF generation logic using jsPDF
- **Created:** `src/lib/utils/jspdf-stub.ts` - Fallback stub for when jsPDF not installed
- **Created:** `src/lib/components/ProofButton.svelte` - Fixed floating action button
- **Created:** `src/lib/stores/calculator-data.svelte.ts` - Store for calculator snapshots (alternative approach)

### Files Modified:
- **Modified:** `src/routes/+page.svelte` - Added ProofButton component
- **Modified:** `package.json` - Intended to add jspdf dependency (see notes below)

### Features:
- Fixed floating button at bottom-right of screen (mobile-safe with safe-area-inset)
- Generates professional PDF "proof sheet" with:
  - Header: "PaveRate — Field Calculation Proof"
  - Timestamp
  - Job Setup Parameters section (width, thickness, spread rate, machine, etc.)
  - Calculator sections for each tool with formulas and GDOT references
  - Footer on all pages: "Generated by PaveRate (paverate.com)"
- Dynamic import of jsPDF (only loaded when needed)
- Fallback stub if jsPDF not available (shows alert)
- Filename format: `paverate-proof-YYYY-MM-DD.pdf`

### PDF Contents:
1. **Job Setup** - All parameters from JobSetupBar
2. **Spread Rate Calculator** - Target rate formula and verification template
3. **Feet Left Today** - Remaining distance calculations
4. **Tonnage to Order** - Material ordering with waste allowance
5. **Tack Rate** - GDOT Table 2 rates and gallons calculation
6. **Stick Check** - Compaction factor formula

## Installation Notes

Due to npm registry connectivity issues in the build environment, the `jspdf` dependency could not be installed during implementation. To complete the setup:

```bash
npm install jspdf
```

The code includes a fallback stub (`jspdf-stub.ts`) that allows the build to succeed without the package, but displays an alert when the export button is clicked.
