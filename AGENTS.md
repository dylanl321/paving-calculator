# AGENTS.md

## Learned User Preferences

- Never add `Co-authored-by: Cursor` (or any Cursor) trailers to commit messages; the user wants commit messages fully clean.
- Do not provide a list of changes in responses.
- Never create fake/sample data or data generators; treat all data as real or captured by real systems.
- Never ask the user to restart development servers.
- For UI work, keep coloring clean, simple, and non-invasive; lean on the brand slate + single yellow accent rather than heavy/decorative color.
- Keep the codebase unified: do not leave unused, duplicated, or over-complex code from multi-agent history; consolidate and remove dead code rather than piling on new files.
- Prefer reframing/inventorying what already exists over adding new features; when asked to "reimagine," audit current capabilities and decide keep/reframe/change rather than building more.
- Keep PaveRate branding visible even when an org sets a custom logo (orgs can customize name/logo/default values, but PaveRate identity stays present).
- Don't state deployment/infrastructure facts (e.g. which branch maps to the `dev.paverate.com` subdomain, what a deploy is serving) as fact from assumption or CLI metadata â€” verify directly (probe the live endpoint/response) before asserting; the user pushes back hard on guessed claims presented as truth. Likewise, don't declare a pipeline/import run a "success" when the review screen still surfaces warnings â€” the user scrutinizes overstated success, so audit each warning honestly (which are real bugs vs benign noise) before claiming it worked. Same with PDF/document data: READ the actual extracted evidence (e.g. `structurer-input/*.evidence.txt`) before theorizing about what a document does/doesn't contain â€” the user pushes back hard ("are you even looking at what the documents show?") when you claim a value "isn't in the PDF" without checking; the value is often present and just unparsed.
- This is a paving app, so the map is roads-only: every line/marker/section must lie on a road. No free-floating off-road points â€” clicks must snap to the route centerline or the real road network. The user repeatedly rejects map editors that let you drop arbitrary points anywhere.

## Map Conventions
